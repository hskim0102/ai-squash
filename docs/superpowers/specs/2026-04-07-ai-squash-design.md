**작성일**: 2026-04-07
**기반 문서**: AI_Squash.md
**버전**: v2.0

## 1. 프로젝트 비전

스쿼시 구력 3년 차 개인 취미 플레이어의 경기 영상을 분석하여, **친근한 트레이너 스타일**의 AI 피드백과 함께 **맞춤형 오늘의 훈련 루틴**을 즉시 제공하는 개인 AI 코치 서비스.

> 핵심 가치: "분석 → 바로 오늘 연습할 드릴 3가지"

---

## 2. 대상 사용자

- **주 사용자**: 개인 취미 스쿼시 플레이어 (자기 발전 목적)
- **로그인 없음**: MVP는 localStorage 기반 익명 동작
- **AI 톤**: 친근한 트레이너 스타일 — 격려와 칭찬을 섞은 동기부여형 피드백

## 4. 페이지 구성 & User Flow

### 4-1. 페이지 트리

```
/ (홈/랜딩)
  └── 히어로: "당신의 랠리를 데이터로 증명하세요"
  └── CTA 버튼: "영상 분석 시작하기" → /analyze

/analyze (분석 페이지)
  ├── Step 1: 영상 업로드 (드래그앤드롭, 최대 1분 권장)
  ├── Step 2: 경기 기록 입력 (선택)
  │     ├── 승/패 토글
  │     ├── 컨디션 슬라이더 (1~5, 😴→😊→🔥)
  │     └── 메모 텍스트 필드
  ├── Step 3: AI 분석 중 (로딩 — 공 튀는 애니메이션)
  └── Step 4: 분석 결과 3-zone 레이아웃
        ├── 상단: 비디오 플레이어 (타임스탬프 클릭 시 해당 구간 점프)
        ├── 중단: AI 피드백 카드
        │     ├── 탭 1: 칭찬할 점 (초록 하이라이트)
        │     └── 탭 2: 개선할 점 (타임스탬프 링크 포함)
        └── 하단: 오늘의 훈련 루틴 카드 슬라이더 (3~5개 드릴)

/history (기록실)
  ├── 과거 분석 리스트 (날짜 / 가장 잘한 점 한 줄 요약)
  ├── 실력 레이더 차트 (정확도 / 파워 / 활동량)
  └── 연속 훈련 스트릭 배지 (3일 / 7일 / 30일)
```

### 4-2. 사이드바 네비게이션

```
🏠 홈
🎬 분석하기
📋 기록실
```

## 5. 핵심 기능 상세

### 5-1. 영상 업로드

- 드래그앤드롭 + 클릭 업로드 지원
- 최대 1분 클립 권장 (UI에 안내 문구 표시)
- 업로드 전 미리보기 썸네일 표시

### 5-2. 경기 기록 입력 (선택 사항)

- 승/패 토글 버튼
- 컨디션 슬라이더 (1~5, 이모지로 시각화)
- 메모 텍스트 필드 (자유 입력)
- 입력 데이터는 Gemini 프롬프트 컨텍스트로 주입 → 더 개인화된 피드백

### 5-3. AI 피드백 카드

- 친근한 트레이너 말투로 출력 (격려 + 기술적 조언 혼합)
- **칭찬할 점** / **개선할 점** 탭으로 명확히 구분
- 개선할 점에는 타임스탬프 링크 제공
  - 예: `00:15` — "백핸드 임팩트 위치를 조금 앞으로 당겨보세요! 거의 다 됐어요 💪"

### 5-4. 오늘의 훈련 루틴 카드 (핵심 신규 기능)

- Gemini가 분석된 약점에 맞춰 3~5개 드릴을 동적 생성
- 각 드릴 카드 구성:
  - 드릴명 (예: "백핸드 스윙 섀도우 연습")
  - 소요 시간 (예: 10분)
  - 난이도 배지 (쉬움 / 보통 / 어려움)
  - 짧은 설명 (2~3줄)
- 카드 슬라이더 형태 (좌우 스와이프)
- 각 드릴에 "완료" 체크 가능
- "루틴 저장" 버튼 → localStorage에 저장

### 5-5. 기록실

- 과거 분석 리스트: 날짜 / 승패 / 가장 잘한 점 한 줄 요약
- 실력 레이더 차트 (Recharts): 정확도 / 파워 / 활동량
- 연속 훈련 스트릭 배지 (가벼운 동기부여 요소)

## 7. Gemini 연동 구조

```
영상 파일 + 경기 기록 (승패/컨디션/메모)
    ↓
Next.js API Route: POST /api/analyze
    ↓
Gemini 1.5 Pro
  - 역할: "친근한 스쿼시 트레이너"
  - 출력 포맷: 구조화된 JSON (단일 응답, 스트리밍 아님)
    {
      praise: string[],
      improvements: { timestamp: string, message: string }[],
      drills: { name: string, duration: string, difficulty: "쉬움"|"보통"|"어려움", description: string }[],
      skills: { accuracy: number, power: number, activity: number }  // 0~100, 레이더 차트용
    }
    ↓
클라이언트에서 로딩 상태 표시 후 결과 렌더링
(Vercel AI SDK 스트리밍은 텍스트 응답에만 적용 — JSON 구조 파싱 안정성 우선)
```

### 프롬프트 설계 원칙

- 역할 고정: "당신은 친근하고 열정적인 스쿼시 트레이너입니다."
- 경기 컨텍스트 주입: 승패, 컨디션, 메모를 프롬프트 앞부분에 포함
- 출력 언어: 한국어
- 칭찬 먼저, 개선점 나중 순서로 구성 (동기부여 유지)
